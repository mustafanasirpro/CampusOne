# Contributing to CampusOne

## Branch Strategy

- main contains stable code.
- Work is completed in small, reviewable commits.

## Commit Style

Examples:

- feat: add JWT authentication
- fix: resolve circular dependency
- docs: update backend development status
- refactor: simplify security configuration

## Code Style

- Keep code beginner-friendly.
- Prefer constructor injection.
- Do not use Lombok.
- Do not use MapStruct.
- Write clear JavaDocs where helpful.
- Keep modules independent.

## Development Workflow

1. Pull latest changes.
2. Implement one logical task.
3. Run tests.
4. Commit with a meaningful message.
5. Push to GitHub.
